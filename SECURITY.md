# Security Policy

## Project context

TauTerm is a personal open-source project maintained on spare time. There is no dedicated security team, no SLA, and no bug bounty program. That said, security issues are taken seriously and will be addressed as a priority within the constraints of a solo maintainer.

## Reporting a vulnerability

**Do not open a public issue for security vulnerabilities.**

Please use [GitHub Security Advisories](https://github.com/Bawycle/tau-term/security/advisories/new) to report vulnerabilities privately. This keeps the details confidential until a fix is available.

Include in your report:
- A description of the vulnerability and its potential impact
- Steps to reproduce or a proof of concept
- The version or commit affected

## What to expect

This is a spare-time project. I will acknowledge reports when I can and fix confirmed vulnerabilities as a priority, but I cannot commit to a specific response timeline. If you have not heard back after a few weeks and wish to disclose publicly, that is reasonable — please mention in your report that you intend to do so.

## Supported versions

Only the latest release is maintained. Older versions do not receive security backports.
